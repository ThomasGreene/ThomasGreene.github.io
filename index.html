Git & Github

There is a difference between "git" and "github".

Upload everything to Github. Shows progress and helps us get in habit.

3 Commands:

  1. Git add:
    • Adds untracked file(s) to an upcoming commit
    • You do not have to add 1:1. Just what you want to add for the next commit.
  2. Git commit
    • Will stage the changes that were made, and added, to be commited or logged.
  3. Git push
    • Commits are only local. You have to "push" commits, in order to, have them on a different system.

Git Log will show a history of all commits for a project. It, in theory, should show you the history/building blocks of the project.

Commit often!You can't over-commit.

Git Branches

    No more making changes on master!
    You must:
  1. Changes go to branch
  2. Push to branch
  3. merge to master
  4. Pull from master
    How to create a branch:
  1. Git checkout -b [filename] ---- name you want the branch to be
  2. Git checkout [filename] ---- switches between branches you've made and the master.
  3. Git push origin [filename] ---- name of branch
    Pulling a branch:
  1. Create a pull request.
  2. Create message to your team that explains your actions.

Different realities. Branch does not effect master until merged with master.

DO NOT WRITE CODE ON MASTER.
    How to Delete Branch From Terminal:
  1. Get approval from two teammates in order to merge.
  2. After merge. Delete branch from Github.
  3. Git branch -D [filename] ---- Delete's branch from terminal
  4. Git branch ---- see if the branch deleted

Git pull origin master ---- pulls master branch from remote master and merges it to local master. They should both sync up.

    How to Clone Repo:
  1. Go into directory where you would like the repo to clone. ---- you do not have to perform "Git init" because the git clone will pull down the .git file and will make the local git for you.
  2. Make sure that you are in the directory where you want the clone to go. NOT THE DIRECTORY WHERE YOU WANT THE FILES TO GO ---- because the clone will pull in the whole directory.
  3. Go into repo on github that you are clonig
  4. Git clone [paste ssh key or url] ---- url or ssh key of rpo you are cloning

Just because you make a change to an old pull request, does not mean you have to make a new pull request.

"Git merge master" ---- merges new master with branch.
(if there are lines that are unable to merge you will receive a conflict message.)

You can accept "Current Change"; "Incoming Change"; or "Bother Changes"

Day 5

gitignore:

    This should be one of the first files you add

    Sometimes there are things that we do not want to push up in git. We want it to be ignored.

  1. Create a ".gitignore" file in the root folder of the project. ---- touch .gitignore
  2. add any files that you want to be ignored.
  3. You can also ignore whole directories. Just add "[directory name]/".
    If there is only one file that you want to ignore within a directory add "[directory name]/[filename]"
  4. One line per ignore. ---- in the .gitignore file

Fetching

    Git Fetch --all:

    Will pull all the branches in a repo.

  1. If you have to pull new updates to a branch git pull origin [branch name]

    2. Branches will be in the .git directory. Once you "checkout" into the branch, it will pull it out. ---- "git branch" command will not show list of branches until you have gone into the branch

  2. After you fetch branches, go into the branch with git checkout [branch name]
  3. After you are done reviewing the branch DO NOT edit on that branch. DON'T WRITE OR FIX SOMEONE ELSE'S CODE
  4. Checkout to branch master

Fixed Mindset vs. Growth Mindset

    Fixed Mindset:

  • "I don't know."

  • Belief that abilities and talents are fixed at birth and cannot change. Sees any challenge as threat to self-image and rigid abilities.

    Growth Mindset:

  • "I don't know, yet."

  • Every challenge is seen as a learning opportunity, and every talent can be grown through feedback, effort, and effective strategies.

  • Put away your ego, to learn.

"We learn and grow through failure, not success."

  1. Hunger to get better everyday(Mastery)
  2. Ability to communicate across tearms (Empathy)
  3. Wants to solve problems that help people (Purpose)
  4. Requires minimal management (Autonomy)
